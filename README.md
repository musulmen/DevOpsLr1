## 1. Цель работы

Целью работы является изучение работы с программным обеспечением Docker для автоматизации развертывания и управления приложениями в средах с поддержкой контейнеризации.

## 2. Ход работы

Сначала мы изучили теоретическую часть по Docker и docker-compose, разобрались с их архитектурой, 
назначением контейнеров и образов, а также изучили основные команды для управления ими. 
После этого мы установили Docker Engine и docker-compose на Ubuntu 20.04 и проверили,
что сервис работает корректно. Затем мы создали Dockerfile для нашего веб-приложения app.py,
настроили проброс порта 1234 и собрали на его основе образ Docker. После этого мы запустили контейнер
с приложением и убедились, что по адресу http://localhost:1234/ возвращается сообщение "Привет мир!".
Далее мы создали файл docker-compose.yml для развертывания приложения вместе с базой данных PostgreSQL,
настроили проброс порта и возможность подключения дополнительных сервисов. Наконец, мы запустили стек 
контейнеров через docker-compose и проверили, что приложение успешно подключается к базе данных.
